EleNa: Elevation-based Navigation
Current routing systemsâ€™ shortest path is not necessarily the shortest path with elevation gain considered. The goal of this project is to design a navigation system that takes the elevation gain of a route into account. We consider two features of elevation gain - maximum elevation gain and minimum elevation gain. The software is written in Python and HTML, CSS where Python Flask App server was used to generate the route.

Fields Description:

  • Source: User can enter the starting location
  • Destination: User can enter the ending location
  • Algorithm: Select Algorithm from the drop-down menu for calculating the route. Currently supported algorithms are A star and Dijkstra.
  • Percentage of elevation gain: It limits the resultant route to x% of the shortest path
  • Strategy of elevation gain: It lets the user choose to see the route with minimum or maximum elevation gain.

Steps to Run the Software:

  • Enter the source.
  • Enter the destination.
  • Select Algorithm from the drop-down menu for calculating the route.
  • Set the percentage of elevation gain using the slider
  • Select whether you want minimum or maximum elevation gain.
  • Click of search
  • You will be able to see the route in red which should be followed according to the values entered in the fields.
